# Build. Learn. Share. Repeat.

This is the core motto and operating loop behind everything Amit does.

- **Build** — ship real software, products, and systems rather than only theorize about them.
- **Learn** — every build generates knowledge, including what failed and why.
- **Share** — write, speak, and teach what was learned so it compounds beyond one project.
- **Repeat** — the loop, not any single project, is the point.

## Why this matters

Technologies, employers, and titles change over a 27+ year career. A loop-based philosophy survives all of them: it explains behavior in 1997 (learning web development) as well as behavior in 2026 (learning AI orchestration) without needing to be rewritten.

## How to apply

This motto should recur across generated artifacts (resume, website, talks, newsletter) as the connective thread between otherwise disparate eras of the career. See [career](../career/) for the eras this loop threads through, and [systems-thinking.md](systems-thinking.md) for the deeper identity this loop serves.
