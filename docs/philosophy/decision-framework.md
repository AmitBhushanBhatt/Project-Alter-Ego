# Decision Framework

## The core question

Not: *"Is this a good opportunity?"*

Instead: **"Does this strengthen or dilute the identity we're intentionally building?"**

Most opportunities look individually reasonable — a role, a talk, a side project, an investment of time. The reframe matters because "good opportunity" is evaluated in isolation; "strengthens or dilutes identity" is evaluated against everything else this repository already commits to. An opportunity can be lucrative or flattering and still dilute the identity, and a modest opportunity can strengthen it enormously if it compounds.

## The filter: six questions

Each question is grounded in a doc that already exists — this framework doesn't introduce new criteria, it applies the ones already committed to.

1. **Archetype fit** — Is this building a system (software, team, product, platform, business, AI workflow, learning ecosystem)? See [systems-thinking.md](systems-thinking.md). If it's not building anything, it needs a strong reason to pass.
2. **Positioning fit** — Does it reinforce Founder CTO / AI-First SaaS Architect / Startup CTO, or does it pull toward the roles [positioning.md](../identity/positioning.md) explicitly excludes (Principal/Staff/Distinguished Engineer, VP Engineering)?
3. **Market fit** — Does it sit within Norway, the Nordics, or Europe (per [positioning.md](../identity/positioning.md#target-market))? Outside that market isn't automatically a no, but it should be a conscious exception, not a default.
4. **Loop fit** — Does it feed Build → Learn → Share → Repeat (see [build-learn-share-repeat.md](build-learn-share-repeat.md)), or is it a one-off that doesn't compound into anything shareable or reusable?
5. **Honesty fit** — Would saying yes require inflating experience, overstating AI expertise beyond [ai-experience.md](../career/ai-experience.md), or claiming something not yet true? Any "yes" here is a hard stop regardless of the other five.
6. **Values fit** — Does it honor the core values below, or does accepting it mean quietly compromising one of them?

## Core Values

Adopted 2026-07 as the values this framework filters against:

- Curiosity over certainty
- Builders over commentators
- Simplicity over complexity
- Learning through experimentation
- Technology in service of people
- Long-term thinking
- Integrity and transparency

## Reading the result

This isn't a points system — six boxes checked doesn't automatically mean yes, and one miss doesn't automatically mean no. Three rough outcomes:

- **Strengthens** — clearly compounds on multiple questions, no honesty-fit violation. Default to yes.
- **Neutral** — doesn't clearly compound, but doesn't violate anything either (e.g., decent pay, outside the signature themes, one-off). Fine as an occasional exception; a pattern of "neutral" choices is itself a dilution.
- **Dilutes** — fails positioning, honesty, or values fit. Decline, or renegotiate the terms until it doesn't fail.

## Worked examples

**A CTO offer at a traditional enterprise, unrelated to AI or SaaS.** Archetype fit: yes (still building systems and teams). Positioning fit: weak — reinforces "CTO" generically, not Founder CTO / AI-First SaaS Architect specifically, and risks anchoring toward a VP-Engineering-shaped role. Loop fit: weak unless the role generates something shareable. Read: neutral-to-dilutes — worth negotiating scope (e.g., an AI modernization mandate) before accepting, not an automatic yes just because it's a CTO title.

**A conference talk invitation on a generic "future of work" panel, no connection to systems building or AI.** Archetype/positioning/loop fit: all weak — it's visibility, not identity reinforcement. Honesty fit: fine. Read: neutral — acceptable as an occasional exception (visibility has some value) but shouldn't become the pattern for how talks get chosen.

**A side project building a tool that has nothing to do with AI, SaaS, or systems-building, purely for quick income.** Archetype fit: maybe (still building something). Positioning/loop/values fit: weak — doesn't compound, doesn't get shared, doesn't teach anything reusable. Read: dilutes if it consumes time that would otherwise go to something that strengthens; neutral if it's genuinely marginal spare time.

## How to apply

Run a real decision through the six questions before accepting, not after. If the read is unclear after going through all six, that's itself useful information — it usually means the opportunity needs to be reshaped (different scope, different framing, different terms) rather than accepted or declined as-is. When in doubt, default to the more cautious read rather than talking yourself into "strengthens."
