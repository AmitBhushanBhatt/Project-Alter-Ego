# Website Strategy

## The website is not version 1.0

This repository is version 1.0. The website is the first generated artifact built from it — see [ADR-0001](../../adr/ADR-0001-knowledge-repository-first.md). If the website ever contains a fact that isn't traceable back to a source domain, that's a process violation, not a shortcut.

## What the website must communicate

- Thinking
- Projects
- Experiments
- Architecture
- Ideas

Not merely employment history. A visitor should leave understanding how Amit reasons (see [philosophy](../philosophy/)), not just what roles he's held.

## How to apply

Website copy should be generated from, and link conceptually back to, the source domains: [philosophy](../philosophy/) for how he thinks, [career](../career/) for what he's built, [identity](../identity/) for positioning. Do not draft website content directly without checking it against those domains first — see the "link, don't restate" rule in [architecture.md](../knowledge/architecture.md). Website implementation itself only starts once the identity, philosophy, career, and knowledge domains are stable (per the ChatGPT handover's immediate backlog).
