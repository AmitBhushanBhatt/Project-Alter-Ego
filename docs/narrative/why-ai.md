# Why AI

Not because it's the trend. Because it's the current era's most interesting raw material for building systems that create leverage — the same reason SharePoint mattered in 2010 and Azure integration mattered in 2020. AI is a medium, not an identity; see [systems-thinking.md](../philosophy/systems-thinking.md) for the fuller argument that the underlying identity was never about any one technology.

What's actually interesting about this era specifically: RAG turns static documentation into something queryable, voice interfaces collapse the distance between intent and action, and AI-assisted engineering changes the unit economics of building software at all — a small team can now ship what used to need a much larger one. That's the systems-builder question again, just with new materials: what leverage does this technology actually create, and for whom?

Deliberately not claimed: production-scale AI expertise, or "AI thought leader" status. The honest position — experimental, hands-on, building real things with Azure OpenAI, RAG, voice assistants, and AI-assisted engineering practices, while openly still learning LangSmith and the wider orchestration ecosystem — is in [ai-experience.md](../career/ai-experience.md), and this essay doesn't get to say anything stronger than that document supports.

Use for the website's Ideas/AI Lab framing and any "why did you pivot to AI" question. If this ever needs to say more than `ai-experience.md` supports, update that document first — this essay follows it, not the other way around.
