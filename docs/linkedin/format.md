# LinkedIn Format

LinkedIn differs from the resume in tone (first person, narrative) and length (headline + About section carry more weight than line-item experience), but sources from the same domains — see [architecture.md](../knowledge/architecture.md)'s link-don't-restate rule. Nothing here duplicates [resume/format.md](../resume/format.md); where they'd say the same thing, this doc only notes the difference.

## Sections

1. **Headline** (LinkedIn's 220-character field) — a denser version of the resume headline. Source: [positioning.md](../identity/positioning.md).
2. **About** — first-person narrative version of the resume summary, built around the [build-learn-share-repeat](../philosophy/build-learn-share-repeat.md) loop. This is the one section that's meaningfully different from the resume: LinkedIn rewards a story, a resume rewards a scan.
3. **Experience** — same entries and same constraint as [resume/format.md](../resume/format.md#sections-in-order) section 4: company names, dates, and titles are required before this can be filled in.
4. **Featured / Activity** — links out to actual content once [Tier 3](../../BACKLOG.md#tier-3--content--thought-leadership) (newsletter, case studies, talks) exists. Empty until then — don't fill it with placeholder content.

## Hard constraints

Same as the resume: no fact here is a source of truth, no exaggeration beyond [ai-experience.md](../career/ai-experience.md), no optimizing toward the roles excluded in [positioning.md](../identity/positioning.md#positioning-guardrails).
